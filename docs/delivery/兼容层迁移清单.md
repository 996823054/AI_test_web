# 兼容层迁移清单

**日期：** 2026-07-15  
**分支：** `refactor/platform-architecture-reorg`

## 仍保留的兼容层

| 兼容入口 | 权威目标 | 删除条件 |
|---|---|---|
| `app.config` / `app.database` / `app.core.*` | `app.platform.*` | 全库不再 import 旧路径 |
| `app.routers.*` / `app.modules.register_routers` | `app.api.*` | 启动入口与测试均改用 `app.api` |
| `app.services.*` / `app.models.*` / `app.schemas.*` | `app.domains.*.*` / `app.ai.*` / `app.platform.*` | 后端与测试零调用 |
| `app.skills.*` | `app.ai.skills.*` | 后端与测试零调用 |
| `/api/ai/documents*` | `/api/requirements/documents*` | 前端与外部调用完成切换 |
| `frontend/src/legacy/` | `frontend/src/modules/*` | legacy 目录无页面引用 |

## 本轮已退役

- 默认路由不再装配 `midscene-ios`（需 `FEATURE_MIDSCENE_IOS=1`）
- 删除无调用的空壳 `app.modules.{cases,ai,api_management,changelog,reports}`
- Vision 标注为 `experiments/vision`，不进默认主路由

## 验收门禁

- 仓库治理测试
- 后端单元 / 契约 / 平台壳 / 双前缀 / 执行报告缝测试
- 前端 vitest + build
- 架构守卫：Router 不直连 Runner/TestExecutor；APIRunner 不写正式资产
