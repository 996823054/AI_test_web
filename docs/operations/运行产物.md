# 运行产物说明

## 不进入 Git 的内容

- `.logs/`、`logs/`：本地服务日志。
- `*.db`、`*.sqlite`、`*.sqlite3`：本地数据库。
- `uploads/`、`backend/uploads/`：用户真实上传文件。
- `artifacts/`：截图、XML、报告和执行证据。
- `__pycache__/`、`*.pyc`：Python 缓存。
- `dist/`：前端构建产物。
- `node_modules/`、`.vite/`、`.vitest/`：依赖和前端工具缓存。
- `.idea/`、`.superpowers/`：本地 IDE 与代理工具状态。

这些目录可由应用或工具重新创建。停止 Git 跟踪不等于删除本地数据；不得为了清理仓库而物理删除用户数据库、上传文件或报告。

## 测试资料

需要长期版本控制的样例必须放到 `docs/reference/` 或测试 fixture 目录，并使用脱敏、可复现内容。真实上传目录不能兼任 fixture 目录。

## 检查方式

```bash
python -m unittest tests.test_repository_governance -v
git status --short --ignored
```

若治理检查报告运行产物仍被跟踪，应先确认文件属于可再生或本地数据，再使用 `git rm --cached` 停止跟踪；不要删除本地原文件。

## 备份与清理

- 清理 SQLite、上传文件和执行产物前必须由数据所有者确认并先备份。
- 报告保留策略以 PRD 和系统设置为准，默认保留期不能替代人工备份判断。
- 日志和导出内容必须脱敏，密钥不得写入普通日志。
