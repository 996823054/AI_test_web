# RC-10 验收证据

## 验收范围
- 需求中心：解析门禁、问题项生命周期、需求树、原文下载、关系视图、影响提示。
- Case 中心：正式 case 契约、AI 草稿人工确认、废弃治理、执行隔离、负样本闭环。
- AI 闭环：拒绝样本脱敏入库、RAG 负样本索引、Phoenix 门禁、变更反向预警。

## 自动化验证命令
```bash
cd backend
python -m unittest discover -s tests -p 'test_*.py' -v
```

```bash
npm test --prefix frontend
npm run build --prefix frontend
```

## 手动验收清单
- 上传存在阻断问题的需求文档，确认不能入库或生成 case，并展示阻断问题清单。
- 对问题项执行修改、采纳 AI 建议、忽略留痕、重新检查，确认修订版和待办状态正确变化。
- 在需求树挂载/移动文档，确认节点文档计数、关系视图、原文下载可用。
- 手动创建 case 时选择 API 归属，确认无归属 case 被拒绝。
- 接受、拒绝、合并 AI 草稿，确认只有人工确认后才进入正式 case。
- 废弃 case 时填写分类、原因和必要关联，确认默认列表隔离废弃 case，执行中心不会执行废弃 case。
- 拒绝 AI 草稿后再次同步相似草稿，确认进入 `check_failed` 并产生高风险待办。
- 对存在 `FLAKY` / `STALE_LOCATOR` 废弃历史的接口查看变更预警，确认前台可展示风险建议。
