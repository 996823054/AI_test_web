# ADR-0003：变更 API 命名

**状态：** 已接受
**日期：** 2026-07-15

## 决策

变更中心的规范资源前缀使用 `/api/changes`，领域名称使用 `changes`，核心对象使用 `ChangeRecord`。

当前代码中的 `/api/changelog` 和 `Changelog` 在后续代码迁移期间保留为兼容入口，不在本批次删除或改变响应。新功能不得继续扩展旧命名；兼容入口只转发到新应用服务，待调用方迁移完成并有契约测试后再移除。

## 原因

`changes` 与 PRD 的“变更中心”和 `ChangeRecord` 一致，也避免把跨模块影响分析误解为单纯日志。兼容策略保证当前调用方和数据库不因文档重组发生行为变化。
