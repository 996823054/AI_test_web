# ADR-0005：执行状态枚举

**状态：** 已接受
**日期：** 2026-07-15

## 决策

统一执行任务的规范状态为：

```text
pending
running
passed
failed
error
cancelled
need_review
```

- `failed`：被测业务或断言得到明确失败结果。
- `error`：环境、设备、网络或执行器异常，未形成可信业务结论。
- `cancelled`：用户取消；只保留任务日志，不生成正式质量报告。
- `need_review`：证据不足、AI 门禁或高风险结果需要人工复核。

步骤或可运行检查可使用自身局部状态，例如 `blocked`、`skipped`；它们不能替代任务主状态。现有批次聚合状态 `completed` 暂作为兼容状态保留，不得用于新的单任务状态，后续迁移需根据子任务结果映射为规范状态并保留旧 API 契约。
