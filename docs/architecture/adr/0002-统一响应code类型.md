# ADR-0002：统一响应 `code` 类型

**状态：** 已接受
**日期：** 2026-07-15

## 决策

平台统一响应中的 `code` 使用字符串。成功值为 `ok`；业务错误使用稳定的 `snake_case` 标识，例如 `requirement_check_failed`。HTTP 状态码继续表达传输层结果，`code` 表达业务结果。

```json
{
  "code": "ok",
  "message": "success",
  "data": {},
  "trace_id": "trace-xxx"
}
```

## 兼容性

当前健康检查和现行开发架构已经使用字符串 `code`。旧文档中的数字 `0` 示例不再作为新接口规范；已有非统一响应不在本批次修改，后续迁移时必须保留旧接口兼容并补契约测试。
