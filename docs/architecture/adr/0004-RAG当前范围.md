# ADR-0004：RAG 当前范围

**状态：** 已接受
**日期：** 2026-07-15

## 决策层次

本 ADR 同时记录产品目标和当前代码实施范围。当前实现可以分步演进，但不得把阶段性实现解释为对 PRD 的缩减。

### PRD 产品目标范围

PRD `FR-AI-003` 要求 RAG 覆盖需求、Case、接口、元素、报告、失败分析、变更记录和未来 Bug 数据。`FR-CASE-005.2` 进一步要求被拒 AI Case 经脱敏后写入 `ai_feedback_samples` 向量集合，并在后续生成和门禁中形成负样本闭环。

上述向量负样本闭环是产品目标，不因当前存储或检索实现较轻量而取消、降级或改写。正式实现还必须遵守来源追溯、脱敏、AI 失败显式失败和人工确认边界。

### 批次 1 / 当前代码实施范围

批次 1 只建立仓库治理和文档权威关系，不实现或迁移 RAG 业务代码。

当前代码已具备受控候选片段、关键词粗筛、真实模型语义重排、来源与召回记录，以及关系型存储中的脱敏负样本检索等基础能力；尚未完成 PRD 要求的独立向量集合和完整知识库运营闭环。当前允许继续使用关系型存储和内存候选集作为兼容实现，但该差距必须由后续交付任务关闭。

当候选片段存在且需要语义重排时，模型配置或调用失败必须显式失败，不允许用规则结果冒充 AI 成功。

## 入库边界

产品入库范围完整遵循 `FR-AI-003`，必须包含需求、Case、接口、元素、报告、失败分析、变更记录和未来 Bug 数据。各类数据进入可信上下文前必须满足对应的人工确认、来源追溯和脱敏要求；检查未通过的需求、未确认 AI 输出和未脱敏内容不得进入正式知识范围。

当前实现只覆盖上述产品范围的子集，主要包括人工确认的需求、Case、接口、报告摘要和脱敏反馈；元素、完整失败分析、变更记录及未来 Bug 数据尚未完整接入。未实现项属于待关闭的实施差距，不代表从产品入库范围中移除。

向量存储的具体技术选型和迁移方案属于后续独立架构决策；这只延后实现方式，不改变 `FR-AI-003`、`FR-CASE-005.2` 及其负样本闭环目标。
