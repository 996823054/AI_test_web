# AI Testing Platform Context

## Domain Language

### Module-complete delivery

The platform is planned by complete functional modules rather than staged staged delivery labels. Development should treat each confirmed functional module as a complete delivery unit.

A module-complete delivery must include:

- User-facing page or workflow entry.
- Required data model and API contract.
- Service-level business orchestration.
- Evidence, status, failure handling, and human-confirmation boundaries where applicable.
- Verification steps or tests that prove the module works end to end.

The baseline business chains are:

- Requirement document -> AI case draft -> tester confirmation -> case asset.
- API case -> single-case execution/debug -> report.
- Android device/mobile path -> device recognition -> mobile case editing/execution evidence.
