# 计算器应用 PRD

版本：v1.0  
文档状态：用于 AI 需求解析能力测试  
适用对象：产品、研发、测试、AI 需求解析服务  
最后更新：2026-05-21

## Problem Statement

用户在日常学习、办公和简单财务场景中，需要一个可快速完成基础四则运算、百分比计算和历史记录查看的轻量计算器。当前目标不是构建专业科学计算器，而是验证 AI 需求解析服务能否从结构化 PRD 中识别功能需求、边界规则、异常处理、验收标准和待确认问题。

## Goals

- 提供加、减、乘、除四则运算能力，支持整数和小数。
- 提供百分比计算、正负号切换、清除、退格和连续运算能力。
- 提供最近 20 条计算历史记录，用户可以查看和复用历史表达式。
- 对除以 0、表达式不完整、数字长度超限等异常给出明确提示。
- 让需求解析系统能识别 `FR-CALC-*` 功能需求、验收标准和待优化问题。

## Non-goals

- 不支持三角函数、对数、指数、开方等科学计算能力。
- 不支持货币汇率换算、单位换算和图形绘制。
- 不支持用户账号、云同步和多设备历史记录同步。
- 不要求实现复杂表达式解析，例如括号嵌套、函数调用或变量定义。
- 不要求接入真实支付、账本或财务系统。

## Personas / Users

- 普通用户：希望快速完成日常四则运算。
- 学生用户：希望检查作业中的简单计算结果。
- 测试人员：希望用该 PRD 验证 AI 能否生成完整测试 case。
- AI 需求解析服务：需要从文档中提取可执行需求点、异常规则和待确认问题。

## User Stories

1. 作为普通用户，我想输入两个数字并选择加减乘除运算，以便快速得到计算结果。
2. 作为普通用户，我想连续输入多个运算步骤，以便不用每次都手动清空结果。
3. 作为普通用户，我想在输入错误时退格删除最后一个字符，以便修正数字。
4. 作为普通用户，我想一键清空当前表达式和结果，以便开始新的计算。
5. 作为学生用户，我想输入小数并得到正确结果，以便验证带小数的计算题。
6. 作为普通用户，我想切换当前数字的正负号，以便快速输入负数。
7. 作为普通用户，我想计算百分比，以便处理折扣、比例和税率相关计算。
8. 作为测试人员，我想查看计算历史，以便确认每次运算的表达式和结果。
9. 作为测试人员，我想看到异常提示，以便验证错误场景是否被正确处理。

## Functional Requirements

### 基础输入与显示

- `FR-CALC-001`：系统必须提供数字键 `0-9`，用户点击数字键后，当前输入区应追加对应数字。
- `FR-CALC-002`：系统必须提供小数点输入能力；同一个数字中最多只能出现一个小数点。
- `FR-CALC-003`：系统必须在主显示区展示当前表达式，并在结果区展示最近一次计算结果。
- `FR-CALC-004`：当用户未输入任何内容时，主显示区应展示 `0`。
- `FR-CALC-005`：单个数字最多允许输入 12 位有效数字，不包含小数点和负号；超过限制时系统应阻止继续输入并提示“数字长度超限”。

### 四则运算

- `FR-CALC-006`：系统必须支持加法运算，表达式格式为 `A + B`。
- `FR-CALC-007`：系统必须支持减法运算，表达式格式为 `A - B`。
- `FR-CALC-008`：系统必须支持乘法运算，表达式格式为 `A × B`。
- `FR-CALC-009`：系统必须支持除法运算，表达式格式为 `A ÷ B`。
- `FR-CALC-010`：当用户点击等号时，系统应计算当前完整表达式并展示结果。
- `FR-CALC-011`：系统应按用户输入顺序进行连续运算。例如输入 `2 + 3 × 4 =` 时，若采用连续计算模式，应得到 `20`。
- `FR-CALC-012`：系统应保留最近一次结果，用户继续点击运算符时，应以该结果作为下一个表达式的左操作数。

### 辅助操作

- `FR-CALC-013`：系统必须提供 `AC` 清除键，点击后清空当前表达式、当前结果和错误提示。
- `FR-CALC-014`：系统必须提供退格键，点击后删除当前输入数字的最后一个字符。
- `FR-CALC-015`：系统必须提供正负号切换键，点击后将当前数字在正数和负数之间切换。
- `FR-CALC-016`：系统必须提供百分比键，点击后将当前数字转换为百分比值。例如 `50%` 应转换为 `0.5`。
- `FR-CALC-017`：当用户连续点击多个运算符时，系统应以最后一次点击的运算符为准。例如 `8 + - 3 =` 应按 `8 - 3` 计算。

### 历史记录

- `FR-CALC-018`：系统必须保存最近 20 条成功计算历史，包含表达式、结果和计算时间。
- `FR-CALC-019`：当历史记录超过 20 条时，系统应删除最早的一条记录。
- `FR-CALC-020`：用户点击历史记录时，系统应将该条历史的表达式和结果回填到显示区。
- `FR-CALC-021`：用户可以一键清空所有历史记录；清空前系统应二次确认。

### 异常处理

- `FR-CALC-022`：当用户执行除以 0 时，系统不得崩溃，应展示错误提示“不能除以 0”。
- `FR-CALC-023`：当表达式不完整时，例如 `8 + =`，系统不得计算，应展示错误提示“表达式不完整”。
- `FR-CALC-024`：当计算结果超过 12 位有效数字时，系统应使用科学计数法展示。
- `FR-CALC-025`：当用户输入非法状态，例如多个小数点、连续百分号或空表达式求值时，系统应阻止操作并展示可理解提示。

### 可访问性与体验

- `FR-CALC-026`：所有按钮必须支持键盘操作，数字键、运算符、Enter、Backspace 和 Escape 应有对应行为。
- `FR-CALC-027`：计算结果变化时，系统应在 100ms 内完成显示更新。
- `FR-CALC-028`：错误提示必须在主显示区附近展示，颜色与普通结果区分明显。

## Acceptance Criteria

### AC-001：基础加法

关联需求：`FR-CALC-001`、`FR-CALC-006`、`FR-CALC-010`

Given 用户打开计算器  
When 用户依次输入 `2`、`+`、`3`、`=`  
Then 系统应展示结果 `5`

### AC-002：小数计算

关联需求：`FR-CALC-002`、`FR-CALC-006`、`FR-CALC-010`

Given 用户打开计算器  
When 用户输入 `1.5 + 2.25 =`  
Then 系统应展示结果 `3.75`

### AC-003：除以 0 异常

关联需求：`FR-CALC-009`、`FR-CALC-022`

Given 用户已输入 `9 ÷ 0`  
When 用户点击 `=`  
Then 系统应展示错误提示“不能除以 0”  
And 系统不得写入成功计算历史

### AC-004：表达式不完整

关联需求：`FR-CALC-023`

Given 用户已输入 `8 +`  
When 用户点击 `=`  
Then 系统应展示错误提示“表达式不完整”  
And 当前表达式应保留在显示区，便于用户继续输入

### AC-005：连续运算

关联需求：`FR-CALC-011`

Given 用户打开计算器  
When 用户输入 `2 + 3 × 4 =`  
Then 系统应按连续计算模式展示结果 `20`

### AC-006：百分比计算

关联需求：`FR-CALC-016`

Given 用户输入数字 `50`  
When 用户点击 `%`  
Then 当前数字应转换为 `0.5`

### AC-007：历史记录上限

关联需求：`FR-CALC-018`、`FR-CALC-019`

Given 用户已经完成 20 条成功计算  
When 用户完成第 21 条成功计算  
Then 历史记录中应只保留最近 20 条  
And 最早的一条历史记录应被删除

### AC-008：键盘操作

关联需求：`FR-CALC-026`

Given 用户聚焦在计算器页面  
When 用户按键盘输入 `7`、`*`、`6`、`Enter`  
Then 系统应展示结果 `42`

## Guardrails

- 系统不得在除以 0、表达式不完整或非法输入时生成成功计算历史。
- 系统不得在用户未确认时清空所有历史记录。
- 系统不得将错误提示伪装为计算结果。
- 系统不得因为单次非法输入导致页面崩溃、按钮失效或历史记录丢失。
- 如果 AI 需求解析服务识别出冲突、缺失或待确认项，必须标记为“待优化”，不能直接当作正式功能需求入库。

## Data / API / UI Implications

### Data

- 计算历史字段应包含：`expression`、`result`、`created_at`。
- 错误记录默认不进入成功历史，但可以进入调试日志。
- 本地存储最多保存 20 条历史记录。

### API

- v1.0 不要求后端 API。
- 如果后续支持云同步，需要新增历史记录同步 API，该能力不在当前范围内。

### UI

- 页面应包含数字键区、运算符区、主显示区、结果区、错误提示区和历史记录区。
- 错误提示应靠近主显示区展示。
- 历史记录应按时间倒序展示。

## AI Evaluation Strategy

本 PRD 用于验证 AI 需求解析能力。解析服务至少应完成：

- 正确提取 `FR-CALC-001` 至 `FR-CALC-028`。
- 正确识别 `Acceptance Criteria` 与功能需求编号的映射关系。
- 正确识别异常规则：除以 0、表达式不完整、数字长度超限、非法输入。
- 正确识别非目标，避免生成科学计算器、云同步或单位换算相关 case。
- 正确识别以下“待优化 / 待确认”问题，并在解析结果中高亮。

## Risks / Open Questions

- `待确认-001`：连续运算优先级存在歧义。`FR-CALC-011` 当前要求按输入顺序连续计算，因此 `2 + 3 × 4 = 20`；但部分用户可能预期按数学优先级得到 `14`。该规则需要产品确认。
- `待确认-002`：百分比在复合表达式中的语义未完全定义。例如 `200 + 10% =` 是输出 `200.1`，还是按常见计算器语义输出 `220`，需要确认。
- `待确认-003`：历史记录是否需要持久化到浏览器本地存储，还是仅保留在当前会话中，当前文档未完全明确。
- `待确认-004`：科学计数法展示规则只要求超过 12 位有效数字时启用，但未定义小数精度保留位数。
- `逻辑冲突-001`：`FR-CALC-013` 要求 `AC` 清空当前结果，但 `FR-CALC-012` 要求保留最近一次结果作为下一个表达式左操作数；当用户点击 `AC` 后是否还允许继续复用最近结果，需要确认。
